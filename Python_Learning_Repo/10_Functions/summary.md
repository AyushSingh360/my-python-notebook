# Summary – Functions

## Take‑aways
- **Parameters** receive data; **return** sends result back.
- Use **docstrings** to document behavior.
- Use `def` to create; simple is better.

## Syntax recap
```python
def my_func(arg1, arg2=0):
    return arg1 + arg2
```

## Checklist
- [ ] Check return values.
- [ ] Avoid mutable default args.
- [ ] Use `*args` and `**kwargs` for flexibility.
