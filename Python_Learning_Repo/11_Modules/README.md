# Modules

## 1. Topic definition
A module is a file containing Python definitions (functions, classes) that can be imported. A package is a folder of modules.

## 2. Why modules exist
- **Organization**: Group related code.
- **Reusability**: Use the same logic across multiple scripts.
- **Namespace**: Avoid naming collisions.

## 3. Real‑world usage
- Using `math` or `random` from the standard library.
- Breaking a large `main.py` into `utils.py`, `models.py`, etc.
- Installing 3rd party packages like `requests`.

## 4. Key rules & syntax
- `import module`
- `from module import object`
- `if __name__ == "__main__":` block runs only when executed directly.
