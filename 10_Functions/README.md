# Functions

## 1. Topic definition
A function is a named block of code that performs a specific task and can be reused.

## 2. Why functions exist
- **DRY (Don't Repeat Yourself)**: Avoid copying logic.
- **Modularity**: Break complex problems into smaller, manageable pieces.
- **Abstraction**: Hide implementation details.

## 3. Real‑world usage
- A function to validate an email address.
- A handler for a specific API route.
- A utility to format dates uniformly across an app.

## 4. Key rules & syntax
```python
def name(param1, param2=default):
    """Docstring explaining the function."""
    # body
    return value
```
- Defined with `def`.
- Returns `None` by default if `return` is omitted.
- Supports positional, keyword, `*args`, and `**kwargs`.
